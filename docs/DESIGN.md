# 오늘의 우리 — 설계 문서

서비스 기획부터 개발 로드맵까지의 핵심 의사결정 기록.

---

## 1. 서비스 분석

**한 줄 정의:** 매일 하나의 질문으로 **나를 이해하고(혼자) → 서로를 이해하는(커플)** 일일 리추얼 앱.

**해결하는 문제**
- 장기 커플의 대화 고갈·권태 ("우리 할 얘기 없어")
- 감정·생각을 표현하는 습관 부재
- 기존 커플앱의 높은 진입장벽(둘 다 가입해야 시작)

**핵심 가치 / 성장 퍼널**
```
[혼자] 나를 안다 → [연결] 초대코드 → [커플] 서로를 안다 → [지속] 우리의 기록·통계
```
**해자(moat):** ① 혼자 모드(콜드스타트 해결) ② 자기→관계 성장 서사 ③ 데이터 신뢰

**북극성 지표:** 주간 동반 답변 완료 커플 수 / 주간 혼자 기록 활성일수

## 2. 경쟁 분석

| | SumOne(썸원) | Paired | Between(비트윈) | **오늘의 우리** |
|---|---|---|---|---|
| 핵심 | 커플 1일1질문+캐릭터 | 커플 1일1질문+전문가 | 커플 사적공간(채팅·앨범) | **혼자→커플** |
| 혼자 사용 | ❌ | ❌ | ❌ | ✅ |
| 약점 | 콜드스타트 | 영어권·구독 | 2025 데이터 유실 신뢰 타격 | (인지도 0) |

**포지셔닝:** 3대 경쟁자는 전부 "커플 전용". **"혼자 사용 가능 + 깊은 성찰" 사분면은 비어 있고, 그게 우리 자리.**

> 블라인드 리빌은 SumOne 이 이미 검증한 **필수 기능**(우리 고유 차별점 아님). 진짜 차별점은 **혼자 모드**.

## 3. MVP 정의

**검증 가설:** "혼자서도 쓸 이유가 있고(리텐션), 연인이 생기면 함께 쓰게 된다(전환)."

**스코프:** 혼자 모드는 완본, 커플 모드는 본질(연결·블라인드리빌·우리기록)만 → 퍼널 전체를 얇게 검증.

| In | Out(Post-MVP/Premium) |
|---|---|
| Google/Apple 로그인, 혼자(질문·감정·칭찬·히스토리·그래프), 커플(연결·리빌·우리기록·streak), FCM 2종, 계정삭제 | Kakao로그인, 관계통계 심화, AI 리포트/질문/감정분석, 사진첨부, 채팅, 캐릭터, 다국어 |

**출시 필수:** 인앱 계정삭제(스토어 정책), 개인정보처리방침, Apple 로그인 동시제공, 데이터 신뢰(백업/내보내기).

## 4. 기능 명세 (화면 12종)

`홈 · 기록 · 우리 · 마이` 4탭 + 풀스크린(질문/감정/칭찬/우리기록) + 온보딩.

**전역 규칙**
- 하루 = `dateKey(yyyy-MM-dd, 로컬)`, 질문은 dateKey 결정적 선택
- 감정·칭찬은 **항상 개인**, 질문은 미연결=개인 / 커플=공유(블라인드)

**블라인드 리빌 상태머신**
```
NONE → (내 제출) → ME_ONLY → (상대 제출) → REVEALED
     → (상대 제출) → PARTNER_ONLY → (내 제출) → REVEALED
```
- 수정: ME_ONLY 동안만(상대가 못 봄) / REVEALED 후 잠금
- 무결성: 보안규칙으로 "내가 제출해야 상대 답 read 허용"

## 5. DB 설계 (Cloud Firestore)

```
users/{uid}                         # 프로필, coupleId
  └─ entries/{dateKey}              # 혼자 기록(질문답변·감정·칭찬)
couples/{coupleId}                  # memberUids, members(닉네임 비정규화), streak
coupleInvites/{code}                # 초대코드(만료)
coupleAnswers/{coupleId}/days/{dateKey}     # 공유질문 메타·리빌
  └─ responses/{uid}               # 멤버별 답변(리빌 전 원본, read 게이트)
questions/{questionId}              # 질문 풀
```

**핵심:** `responses/{uid}` 를 별도 문서로 분리 → 보안규칙으로 블라인드 리빌 강제(문서 단위 read 제어).
**인덱스:** `days(revealed, dateKey desc)`, `questions(active, seq)`.
